¿Qué ocurre cuando ejecutas node server.js?
Al ejecutar el comando, Node.js inicializa el proceso de nuestra aplicación de Express, compila el script en tiempo de ejecución y levanta el servidor HTTP en modo de escucha pasiva en la máquina local.

¿Qué mensaje aparece en la terminal?
Aparece el mensaje configurado en el callback del método listen:
"Servidor ejecutándose en puerto 3000"

¿Qué sucede cuando visitas la ruta /estado?
El navegador realiza una solicitud HTTP con el método GET a la dirección 'http://localhost:3000/estado'. El servidor procesa el endpoint y retorna un objeto en formato JSON como respuesta estructurada:
{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}
